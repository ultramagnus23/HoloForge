## Core library (`holomedia/`)

| File | Role |
|---|---|
| `npdd.py` | 1D `NPDDRecorder`: differentiable NPDD reaction-diffusion twin (spectral IMEX, `forward` unrolled + `forward_checkpointed` discrete-adjoint), `MediumParams` dataclass, analytic `small_signal_mtf`/`predicted_cliff` (Eq. 5). Batching support (per-row `D_eff.mean(dim=-1)`) added this session. |
| `diffraction.py` | `kogelnik_de` closed form + `SlabBPM` split-step scalar readout. Batching support (shape-derived plane-wave init) added this session. |
| `optimize.py` | Method registry: `media_in_the_loop`, `media_blind_sgd`, `media_blind_gs`, `oracle_ideal` + batched counterparts (`*_batched`), `psnr`/`psnr_batch`, `diffraction_efficiency`/`_batch`. Seed-init bug fixed this session (`_seeded_init_theta`). **This is the file Phase 2's method registry (M1-M5b) will extend.** |
| `npdd3d.py`, `diffraction3d.py` | 3D (x,y,z) showcase generalizations, kept as separate modules from the 1D pipeline by design (see file docstrings) so the tested 1D path is untouched. Used only by `experiments/showcase_3d.py`. Not part of the paper's main (2D x,z) results. |
| `surrogate.py` | Small 1D CNN `DnSurrogate`, offline-trained cheap gradient source. Used only by `experiments/ablation_gradients.py`. |
| `__init__.py` | Exports the 1D `optimize.py`/`npdd.py`/`diffraction.py` API only. 3D and surrogate modules are imported directly by their scripts, not re-exported. |

## Divergences from the master prompt's assumed file list

The master prompt's file list is accurate for everything it names, with these corrections/additions:
- `holomedia/optimize.py`'s batched functions are named `media_in_the_loop_batched`, `media_blind_sgd_batched`, `media_blind_gs_batched`, `oracle_ideal_batched` — not a single generic "batched mode" flag.
- Two GPU scripts not in the master prompt's list also exist and are relevant to Phase 1's manifest system: `experiments/gpu_npdd_mesh_convergence_sweep.py` and `experiments/gpu_bpm_wavelength_sweep.py` (already run once, single-seed, results committed under `results/gpu_reruns/`).
- `experiments/_gpu_common.py` (device/checkpoint/timing helpers) is the closest existing precursor to Phase 1.1's manifest system — Phase 1 should decide whether to extend it or supersede it, not duplicate it.
- `experiments/f2_f3_recovery.py` (CPU) and `experiments/gpu_f2_f3_recovery.py` (GPU, batched) both exist; the master prompt's Phase 3 experiments (E1-E6) should build on the GPU one's batched method calls, not the CPU one's per-item loop.
- `holomedia/npdd3d.py`, `diffraction3d.py`, `surrogate.py`, and `experiments/showcase_3d.py`, `ablation_gradients.py` were not in the master prompt's list but are real, already-run components the current tex cites (Fig. E/F, the 3D showcase paragraph in Limitations).

## Flagged issues (not yet fixed, for Phase 1+ triage)

1. **Config files unused**: `configs/media/*.yaml` are never loaded programmatically. Either wire them into the manifest system (Phase 1) so E1-E5 can target real medium families by name, or state plainly that `MediumParams()` defaults are the only medium actually swept.
2. **`results_f1.pt` not committed** — breaks the "all code, configurations, and result JSONs... available at github.com/..." claim in the tex's Code and Data Availability section for Fig. 1 specifically.
3. **No dead code found** — every module in `holomedia/` is imported by at least one experiment script or `__init__.py`; every experiment script produces a results JSON that's either cited in the tex or documented as superseded in `CHANGELOG.md`.
4. **Stray log files** (`confirm.log`, `prelim.log`) are untracked by omission, not by `.gitignore` rule — low risk, worth a one-line `.gitignore` addition.
